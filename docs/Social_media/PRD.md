# Le Relief — Social Media Renderer · Product Requirements Document

> **Status:** Draft v1.0 — May 2026  
> **Scope:** `packages/renderer`, `packages/renderer-server`, `src/lib/social/`  
> **Audience:** Engineering, Editorial, Product

---

## 1. Purpose & Context

Le Relief is Haiti's leading digital news outlet with an active presence on Facebook, Instagram, and X/Twitter (43.5 K followers, verified). The renderer is the engine that converts a published article into ready-to-post visuals and captions for every social platform — no manual design work required.

The V1 renderer was ported from EdLight News and produces technically correct output, but the generated captions and image layouts do not match Le Relief's actual posting voice or editorial conventions. This PRD defines what the renderer **must** produce to match the brand's real-world social media behaviour, and what new capabilities are needed.

### 1.1 Problem Statement

| # | Pain Point | Source |
|---|---|---|
| P1 | X/Twitter captions are formatted like Instagram posts — long excerpts that get thread-split into 3+ tweets | X feed analysis: real posts are ≤ 150 chars |
| P2 | Breaking-news tweets don't use the `Flash 🚨\n🇭🇹` prefix that the account's verified followers expect | X feed: pinned + most-engaged posts use this format |
| P3 | Hashtags are always the same 3 generic tags regardless of topic | No topical discoverability; each article should carry category-relevant tags |
| P4 | `#Haiti` (English spelling) is absent; limits international reach | Cross-lingual audience missed |
| P5 | All platforms receive the same caption text; no platform-native tone | Facebook supports longer emotional hooks; WhatsApp wants no hashtags |
| P6 | `keyFacts` on `ContentIntakeInput` was never populated; template selection is coarser than needed | Template intelligence underused |
| P7 | Images are all 1080×1350 regardless of platform; X landscape (1600×900) looks letterboxed | Visual quality on X degraded |
| P8 | No "caricature" content type — a recurring weekly post format for the account | Custom format not supported |
| P9 | No story/reel format with animated-feel layouts for Instagram & TikTok | Story assets must be created manually |
| P10 | The renderer-server (`packages/renderer-server`) had its own duplicated caption/hashtag logic, drifting from the main library | Two sources of truth, inconsistencies in cloud-run mode |

### 1.2 Goals

1. **Voice parity** — captions generated by the engine must be indistinguishable from what the editorial team would write manually.
2. **Platform-native output** — each platform gets a caption, image format, and hashtag set designed specifically for it.
3. **Zero config for editors** — the social generator dashboard should "just work" with no manual caption rewriting for 80 % of articles.
4. **Extensibility** — adding a new content type or platform must require changing ≤ 3 files.
5. **Reliability** — the Cloud Run renderer-server and the inline renderer must always produce identical output.

### 1.3 Non-Goals (V2+)

- AI-generated captions (GPT/Claude integration) — deferred to V2
- Scheduling and auto-publishing to X, TikTok, Threads — V2
- Animated video export (MP4/GIF) for Reels/TikTok — V2
- A/B caption testing — V3

**Data flow for one article render:**
1. `article-to-post.ts` — maps `Article` → `{ intake, rawSlides, caption, contentType }`
2. `buildPost()` — validates, selects template, fits text, builds `EnginePost`
3. `renderPost()` — takes screenshots via Playwright (Chromium) → PNG buffers
4. `formatForPlatform()` — per-platform caption adapter produces `AdapterOutput`
5. `uploadSlide()` — buffers pushed to Firebase Storage
6. `socialRepo.upsert()` — `PlatformPostState` persisted in Firestore

## 4. Caption System Requirements

### 4.1 `PostCaption` Structure

```ts
interface PostCaption {
  text: string;       // Full caption (Instagram, Facebook)
  shortText?: string; // Short caption (X, Threads, WhatsApp) — Le Relief voice
  hashtags: string[]; // Category-aware tags (see §4.3)
  cta?: string;       // Article URL for CTA slide
}
```

### 4.2 Platform Caption Targets

| Platform | Caption Source | Max Chars | Hashtags | Notes |
|---|---|---|---|---|
| Instagram Feed | `text` | 2,200 | In `firstComment` (hidden) | Excerpt + URL + CTA |
| Instagram Story | `text` (short) | 2,200 | None | Story overlaid text |
| Facebook Feed | `text` + emotional hook | 63,206 | Yes (inline) | Can be slightly longer; invite engagement |
| X Portrait / Landscape | `shortText` | 280 | 2–3 inline | `Flash 🚨\n\n🇭🇹 {headline}` or `🇭🇹 {headline}` |
| Threads | `shortText` | 500 | 2–3 inline | Same voice as X |
| WhatsApp Status | `shortText` stripped of hashtags | 700 | None | No discovery surface |
| TikTok | `text` | 2,200 | Yes | Same as Instagram |
| LinkedIn | `text` | 3,000 | Yes | More formal tone |
| YouTube Short cover | `text` (short) | 5,000 | Yes | Title + description |

### 4.3 Hashtag Strategy

**Base tags** (always present):
- `#LeRelief` — brand
- `#Haïti` — French-language Haiti tag
- `#Haiti` — English-spelling for international reach
- Language tag: `#Actualités` (fr) or `#News` (en)
- `#Flash` — only on breaking news

**Category tags** (one added when category matches):

| Category slug | Tag |
|---|---|
| `politique` | `#Politique` |
| `securite` / `securite-publique` | `#Sécurité` |
| `economie` | `#Économie` |
| `culture` | `#Culture` |
| `sport` / `sports` | `#Sport` |
| `diaspora` | `#Diaspora` |
| `education` | `#Éducation` |
| `sante` | `#Santé` |
| `environnement` | `#Environnement` |
| `international` | `#International` |
| `justice` | `#Justice` |
| `droits` | `#DroitsHumains` |
| `humanitaire` | `#Humanitaire` |
| `technologie` | `#Tech` |

### 4.4 Short Caption Voice Guide (X / Threads / WhatsApp)

Le Relief's verified X feed uses three distinct micro-patterns:

**Pattern A — Breaking / Flash**
```
Flash 🚨

🇭🇹 {headline — one sentence, no period}
```

**Pattern B — Standard news**
```
🇭🇹 {headline}
```

**Pattern C — Caricature** (weekly visual content)
```
CARICATURE du jour avec Le Relief

Dessinateur : {author name}
```

Rules:
- No URL in X/Threads captions (the image IS the post; URL in first reply is fine)
- No more than 3 hashtags on X
- Never split into a thread for regular news; threads only for explainers > 400 chars
- Use emoji flags sparingly — 1–2 per post maximum

### 4.5 Facebook Caption Voice Guide

Facebook supports longer posts and an emotional tone. The adapter should produce:

```
{headline}

{1–2 sentence excerpt, written for an audience, not a machine}

👉 Lire l'article complet : {url}

{hashtags}
```

The URL must always appear in Facebook captions (link preview may not always attach).

## 5. Visual Template Requirements

### 5.1 Existing Templates

| Template ID | Layout | Slides | Used For |
|---|---|---|---|
| `breaking-news-single` | Single hero, gradient overlay, no carousel | 1 | Breaking / urgent |
| `news-carousel` | Cover + detail(s) + CTA | 3–5 | Standard news |
| `opportunity-carousel` | Branded opportunity layout | 2–4 | Jobs, scholarships |
| `explainer-carousel` | Step-by-step explainer | 3–6 | Analysis, context |
| `quote-stat-card` | Giant stat or pull-quote | 1–2 | Data, quotes |
| `weekly-recap-carousel` | Recap with numbered items | 3–7 | Weekly summaries |
| `data-card` | Chart-style data visualization | 1–3 | Statistics |

### 5.2 New Templates Required

#### 5.2.1 `caricature-card` (Priority: HIGH)

Le Relief publishes a weekly caricature illustrated by Francisco Silva. Current system has no template for this.

**Layout:**
- Full-bleed illustration image (supplied as `imageUrl`)
- Bottom strip: `CARICATURE du jour avec Le Relief` in brand font
- Small credit line: `Dessinateur : {authorName}`
- No headline overlay on the image (art must breathe)
- Footer bar: Le Relief wordmark + `@lereliefhaiti`

**Slides:** 1 (single image)  
**Canvas:** 1080×1080 (square, fits all platforms equally)  
**Content type:** `caricature`  
**Trigger:** `article.contentType === "caricature"` OR `article.tags` includes `"caricature"`

#### 5.2.2 `story-cover` (Priority: MEDIUM)

A vertical (9:16) layout optimised for Instagram Story and TikTok cover.

**Layout:**
- Full-bleed cover image with dark gradient vignette
- Top: date pill + category label
- Centre: headline in large white type
- Bottom: `Lireliefhaiti.com` + wordmark + handle

**Slides:** 1  
**Canvas:** 1080×1920  
**Platforms:** `instagram-story`, `instagram-reel-cover`, `tiktok`, `youtube-short-cover`

#### 5.2.3 `quote-pull` (Priority: LOW)

Styled pull-quote for editorial statements, official declarations, source quotes.

**Layout:**
- Dark background with large typographic quotation mark
- Quote text centred, italic editorial font
- Attribution line: `— {source name}, {role}`
- Category pill: `CITATION`

**Slides:** 1  
**Canvas:** 1080×1080 (square)

### 5.3 Platform Canvas Compliance

All templates must render correctly at the following sizes:

| Platform | Canvas | Aspect | Safe Area |
|---|---|---|---|
| Instagram Feed | 1080×1350 | 4:5 | 90px all sides |
| Instagram Story | 1080×1920 | 9:16 | 120px top/bottom, 90px sides |
| Facebook Feed | 1200×1500 | 4:5 | 100px all sides |
| Facebook Link | 1200×630 | 1.91:1 | 80px all sides |
| X Portrait | 1080×1350 | 4:5 | 92px top/bottom, 90px sides |
| X Landscape | 1600×900 | 16:9 | 70px top/bottom, 90px sides |
| WhatsApp Status | 1080×1920 | 9:16 | 120px top/bottom, 90px sides |
| TikTok | 1080×1920 | 9:16 | 150px top/bottom, 90px sides |

### 5.4 Visual Identity Rules

- **Background palette:** Deep navy (`#020b17`) for news, near-black with red tint (`#150408`) for breaking — never pure black
- **Accent colour:** `#5090E0` (Le Relief blue) — used for category pills, CTAs, rule lines
- **Breaking accent:** `#f43f5e` — only for breaking news category pill and footer rule
- **Wordmark:** `LE` (regular weight) + `RELIEF` (bold, accent colour) — see `brand.ts`
- **Typography:** DM Sans for headlines, Inter for body/captions, Playfair Display for editorial/quote contexts
- **Logo:** Resolve from `BRAND.logoPath` or fall back to wordmark-only; never use a raster that renders blurry at screenshot resolution
- **Footer bar:** Always present on slide 1 and CTA slides; optional on interior slides; shows `sourceLine` left + handle right

## 6. `article-to-post.ts` Requirements

This is the critical translation layer between a `Article` and the renderer engine.

### 6.1 Current Output (V1)

```
slide 1: cover (headline, image)
slide 2: L'essentiel (first 260 chars of body)
slide 3: Lire l'article (CTA)

caption.text: headline + excerpt + 👉 URL
caption.hashtags: ["#LeRelief", "#Haïti", "#Actualités"]
```

### 6.2 Required Output (V2)

**Content-type routing:**

| `article.contentType` / `article.isBreaking` | Template Selected | Slide sequence |
|---|---|---|
| `isBreaking: true` | `breaking-news-single` | 1 slide: hero |
| `contentType: "caricature"` | `caricature-card` | 1 slide: illustration |
| `contentType: "analyse"` / `contentType: "explainer"` | `explainer-carousel` | 3–5 slides: intro, key points, conclusion, CTA |
| `contentType: "data"` | `data-card` | 2–3 slides: data + context + CTA |
| Default | `news-carousel` | 3 slides: cover, detail, CTA |

**Slide content improvements:**
- Slide 2 `body` must use bullet points extracted from `article.body` when the body contains `<ul>/<li>` HTML — not just a raw text slice
- `keyFacts` on `ContentIntakeInput` should be populated with the first 2–3 extracted bullet points
- `sourceLine` should include the author's title/role if `article.author.title` is available

**Caption `shortText` generation:**

```
isBreaking: true   → "Flash 🚨\n\n🇭🇹 {headline}"
caricature         → "CARICATURE du jour avec Le Relief\n\nDessinateur : {authorName}"
default (fr)       → "🇭🇹 {headline}"
default (en)       → "🇭🇹 {headline}" (unchanged; flag works for both languages)
```

**Hashtag generation:**
- Always: `#LeRelief`, `#Haïti`, `#Haiti`, + language tag
- Breaking: add `#Flash`
- Category slug: map to category tag (see §4.3)
- Tags from `article.tags[]`: convert to hashtags if they are known topical terms (e.g. `gang` → `#Gangs`, `diaspora` → `#Diaspora`)

### 6.3 Parity Between `article-to-post.ts` and `renderer-server`

The renderer-server (`packages/renderer-server/src/index.ts`) contains a private `articleToContent()` function that mirrors `article-to-post.ts`. These two **must be kept in sync**. Any logic change in one must be applied to the other.

**Preferred approach (V2):** Extract the shared logic into `@le-relief/renderer` itself as a `articleContentAdapter` export, so the renderer-server can import it directly instead of duplicating it.

## 7. Adapter System Requirements

### 7.1 `xAdapter` — Detailed Spec

```
Input:  post.caption.shortText  (preferred)
        post.caption.text       (fallback)
        post.hashtags[0..2]     (max 3 tags)

Output: {
  caption: string,   // = chunks[0], the first (and usually only) tweet
  thread: string[],  // = [caption] for short posts; multiple chunks for explainers
  meta: { handle }
}
```

Rules:
- `caption` must be the **first tweet text only** — not all chunks joined with `---`
- Thread only triggers if `shortText` itself exceeds 280 chars (rare; explainer threads)
- Reserve 60 chars at end for hashtags
- Never append the article URL to X captions (the image carries the branding)

### 7.2 `facebookAdapter` — Detailed Spec

Facebook's algorithm rewards posts that invite engagement. The adapter should:
- Append the full article URL (not just the CTA text)
- Use `text` (full caption), not `shortText`
- Allow all hashtags
- Optionally append an engagement hook: `"Que pensez-vous ?"` / `"What do you think?"`

This engagement hook is off by default; it can be enabled via `BRAND.config.facebookEngagementHook: true` (to be added to `BrandConfig`).

### 7.3 `whatsappAdapter` — Detailed Spec

- Use `shortText` stripped of all hashtags
- Append the article URL (WhatsApp status is link-friendly)
- No hashtags (no discovery surface on WhatsApp)
- Emoji flag kept (renders well on mobile)

### 7.4 `instagramAdapter` — Detailed Spec

- Caption: `text` (full)
- Hashtags in `firstComment` only (keeps caption clean; improves reach)
- `meta.handle` used in footer bar

## 10. Technical Requirements

### 10.1 Performance

- Full render for all platforms (12 PlatformIds) must complete in < 45 s on a Cloud Run instance (2 vCPU, 4 GB RAM)
- Single-platform re-render must complete in < 10 s
- Chromium browser instance must be shared across all slides in a single render pass (`browser.ts` singleton)

### 10.2 Reliability

- Any single platform failure must not abort the entire render; errors are collected in `warnings[]` and the remaining platforms proceed
- `resetBrand()` must be called in a `finally` block to prevent brand state leaking between concurrent server requests
- WhatsApp sticker (WebP transparent) must hard-fail (not silently produce a corrupt file) if output exceeds 100 KB

### 10.3 Type Safety

- `PostCaption.shortText` is `string | undefined` — adapters must fall back to `text` when absent
- `PlatformCaptionOverrides` must be extended to accept `shortText` overrides per-platform when editorial override is needed
- All exported types from `@le-relief/renderer` must be re-exported from `packages/types/src/index.ts`

### 10.4 Testing

Every template must have a render test asserting:
- PNG dimensions match the platform spec's canvas
- No Playwright error or page crash
- Footer bar is present in the output HTML

Caption adapters must have unit tests asserting:
- `xAdapter` output fits in 280 chars
- `xAdapter` uses `shortText` over `text`
- `xAdapter` for breaking news starts with `Flash 🚨`
- `instagramAdapter` hashtags are in `firstComment`, not in `caption`
- `whatsappAdapter` output contains no `#` characters

### 10.5 Smoke Script

`packages/renderer/src/scripts/smoke.ts` must exercise:
- At least one slide per `TemplateId`
- At least one slide per `PlatformId`
- The caricature template (new)
- Breaking vs. non-breaking short caption format

## 12. Acceptance Criteria

The renderer is considered complete for V2 when:

1. A breaking news article produces: `Flash 🚨\n\n🇭🇹 {headline}` as the X caption, fitting in one tweet without thread-splitting.
2. A caricature article produces a single-slide image with the illustration full-bleed and correct credit line.
3. An explainer article produces an `explainer-carousel` with bullet points extracted from HTML body, not a raw text slice.
4. All 12 platforms produce correctly sized PNG/WebP at the canvas dimensions defined in `PlatformSpec`.
5. Hashtags on a `securite` article include `#Sécurité`; on a `diaspora` article include `#Diaspora`.
6. Instagram caption hashtags appear in `firstComment` only, not in the main caption text.
7. Facebook caption always includes the full article URL.
8. WhatsApp status caption contains no `#` characters.
9. The smoke script completes without errors on a fresh Cloud Run container.
10. The renderer-server and the inline renderer produce byte-identical captions for the same article.
